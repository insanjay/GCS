# Contributing to Gesture Control System (GCS)

Thank you for your interest in contributing.

This document explains the contribution workflow.

---

# Contribution Workflow

## Step 1 — Fork the Repository

Click the **Fork** button on GitHub.

This creates your own copy.

---

## Step 2 — Clone Your Fork

```bash
git clone https://github.com/YOUR_USERNAME/GCS.git
cd GCS
```

---

## Step 3 — Create a Feature Branch

Create a new branch for your changes:

```bash
git checkout -b feature/your-feature-name
```

Examples:

```bash
git checkout -b feature/add-new-gesture
git checkout -b fix/cursor-jitter
```

---

## Step 4 — Make Your Changes

Edit the code.

Test your changes.

---

## Step 5 — Commit

```bash
git add .
git commit -m "Add: short description of change"
```

Use clear commit messages.

Examples:

Add scroll smoothing
Fix click detection bug

---

## Step 6 — Push to Your Fork

```bash
git push origin feature/your-feature-name
```

---

## Step 7 — Create Pull Request

Go to GitHub.

Click:

Compare & Pull Request

Base repository:

insanjay/GCS

Base branch:

master

Compare branch:

your feature branch

Submit Pull Request.

---

# Contribution Guidelines

Please ensure:

* Code follows PEP 8
* Changes are tested
* Commit messages are clear
* Do not commit model files unnecessarily
* Do not include unrelated changes

---

# Types of Contributions Welcome

* Bug fixes
* Performance improvements
* New gestures
* Documentation improvements
* Refactoring

---

# Questions

Open a GitHub Issue for discussion.

---

Thank you for contributing.
